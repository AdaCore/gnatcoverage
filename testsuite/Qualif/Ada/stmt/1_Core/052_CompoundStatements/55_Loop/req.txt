Loop statements (ARM 5.5)

%(req-headline)s

LOOP statements feature a mandatory loop *header* that introduces the
control-flow iteration scheme, and an optional set of exit statements.

In addition to the common requirements that apply to the nested statements,
loop headers that are never reached shall be reported uncovered.

%(tstrategy-headline)s

Check the coverage when:

* the LOOP condition is evaluated to TRUE;
* the LOOP condition is evaluated to FALSE;
* the LOOP condition is not evaluated

Check different syntax forms of LOOP statement:

* FOR loop;
* reverse FOR loop;
* WHILE loop;
* unconditional loop.
