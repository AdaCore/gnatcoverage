Material to demonstrate that the toolset correctly handles potentially
confusing constructs that would fool naive analyzers for various sorts of
reasons.  This includes cases such as multiple statements sharing a line or a
single statement spanning multiple lines.

%(req-headline)s

Statement coverage assessment remains well defined in the presence of code
constructs that could fool simple minded analysis engines.

%(tstrategy-headline)s

Check cases where

* Code indentation could be misleading

* The presence of homonym entities could be misleading

* Multiple statements share the same line

* A single statement spans multiple lines

* Entire sequences are inhibited because nested within a
  conditional statically known to be always False
